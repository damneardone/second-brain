# PARA Methodology

The PARA method is a universal system for organizing digital information. It stands for Projects, Areas, Resources, and Archives—the four primary categories that encompass every type of information you might encounter in your work and life.

## 📂 Projects (Проекты)
A **Project** is a "series of tasks linked to a goal, with a deadline."
- Examples: Complete app design; Write a book chapter; Plan a vacation.
- Key characteristic: It has a concrete "done" state.

## 🌐 Areas (Области ответственности)
An **Area of Responsibility** is a "sphere of activity with a standard to be maintained over time."
- Examples: Health; Finances; Professional Development; Home Maintenance.
- Key characteristic: It doesn't have a deadline, but requires ongoing attention.

## 📚 Resources (Ресурсы)
A **Resource** is a "topic or theme of ongoing interest."
- Examples: Photography; Habit formation; Marketing; Project management.
- Key characteristic: Information that is useful for multiple projects or areas, or just interesting.

## 🗄️ Archives (Архивы)
**Archives** include "inactive items from the other three categories."
- Examples: Completed projects; Areas you are no longer responsible for; Resources you are no longer interested in.
- Key characteristic: Items kept for future reference but not currently active.

---

## 🛠️ Categorization Rules
When categorizing a new note or file, ask these questions in order:
1. Which **Project** will this be most useful for right now?
2. If none, which **Area** does this relate to?
3. If none, which **Resource** does this topic belong to?
4. If it's already finished or irrelevant, move it to **Archives**.

## 🧠 Second Brain Implementation
This Agent uses PARA to route all incoming information (Telegram exports, voice notes, manual entries) into the correct organizational bucket. 
- **Atomic Notes** are routed to the relevant PARA folder.
- **Maps of Content (MOC)** bridge multiple PARA categories to provide high-level overviews.
- **Evolutionary Mutations** happen within these structures to keep the system alive.
